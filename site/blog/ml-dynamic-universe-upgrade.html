ML Enhancement Layer & Dynamic Universe: From 500 to 1,578 Tickers

March 19, 2026 • 15 min read • ML XGBoost Dynamic Universe AI • 中文版

StockQueen's momentum engine already delivers strong risk-adjusted returns through regime switching, VIX filters, ATR trailing stops, and dynamic cash allocation. But we asked a harder question: can machine learning find the offensive edge that pure momentum misses? This post documents two ML experiments—one that failed, one that showed promise—and a dynamic universe expansion that unlocked 4.1x more return from a 3x larger ticker pool.

Why ML? The Missing Offense Gene

StockQueen V5 already has a deep defense stack. Regime switching detects bear markets and rotates to cash or defensive ETFs. VIX filters throttle exposure during volatility spikes. ATR-based trailing stops lock in gains and cut losers. Cash allocation rules prevent over-commitment in uncertain environments.

Defense is solved. But offense—the ability to identify high-growth winners before they break out—is still driven purely by momentum scores. Momentum is powerful, but it is backward-looking by nature. ML can potentially incorporate forward-looking features: earnings acceleration, revenue growth trajectories, institutional flow patterns, and cross-sectional relative strength.

Core thesis: The defense stack handles drawdown protection. ML should focus exclusively on offense—finding the stocks most likely to deliver outsized returns in the next rotation window.

ML-V1: The Wrong Direction

Our first attempt used XGBRegressor with the reg:squarederror objective. The model was trained on 17 features to predict absolute forward returns. The idea was simple: predict which stocks will go up the most, then overweight them.

The results were disappointing. ML-V1 improved Sharpe by +0.06, but cumulative returns dropped by −7.3% compared to the baseline. The model was learning to avoid losers rather than find winners—it was duplicating the defense stack, not complementing it.

ML-V1 Result

Sharpe Delta +0.06
Return Delta −7.3%

Absolute return labels caused the model to optimize for loss avoidance. With 17 defensive features, ML-V1 became a redundant risk filter—the same job the regime switcher and VIX filter already do better.

Root cause: Predicting absolute returns with squared-error loss trains the model to minimize large prediction errors. In practice, this means the model learns to avoid volatile stocks—which are exactly the high-growth candidates we want. The label design was fundamentally wrong for an offensive objective.

ML-V2: Redesigning for Offense

ML-V2 made three fundamental changes based on the V1 failure analysis:

1

XGBRanker with rank:pairwise

Instead of predicting absolute returns, the model learns to rank stocks relative to each other. This is a cross-sectional objective: which stock will outperform its peers in the next window?

2

22 features (5 new offensive features)

Added earnings acceleration, revenue growth momentum, institutional ownership change, short interest ratio, and options implied volatility skew. These features capture forward-looking signals that pure price momentum misses.

3

Cross-sectional z-score normalization

All features are z-scored within each time window, removing regime-dependent scaling and forcing the model to focus on relative positioning.

ML-V2 showed meaningful improvement: Sharpe increased by +0.14 with a return delta of −7.0%. The return cost was similar to V1, but the risk-adjusted improvement was more than double. More importantly, the model showed a clear pattern in where it adds value.

Where ML-V2 Wins

Crash and bear environments. When the market sells off indiscriminately, the ranker identifies stocks with defensive characteristics that the pure momentum score misses. W1 2020: +29.1% vs baseline +8.4% (3.5x improvement).

Where ML-V2 Loses

Strong bull markets. When momentum is king and everything is trending up, the ML layer adds friction. The ranker sometimes rotates away from the highest-momentum names in favor of "better quality" stocks that underperform in pure momentum environments.

Status: Work in progress. ML-V2 is not yet deployed to production. The Sharpe improvement of +0.14 is encouraging, but the −7.0% return cost in bull markets needs to be addressed before the layer is production-ready. The defense stack already handles crashes well, so ML-V2's crash advantage has diminishing marginal value.

Dynamic Universe: From 500 to 1,578

While ML experiments continued, we tackled a different bottleneck: the static ticker pool. StockQueen V5 used a fixed list of 479 tickers, curated manually. This creates two problems: survivorship bias (only today's survivors are included) and missed opportunities (stocks that met momentum criteria historically but were never in the pool).

The dynamic universe approach replaces the static list with a screening funnel that runs at each rebalance. At each rotation window, the system scans all US equities and filters down to those meeting minimum liquidity, market cap, and price history requirements.

Survivorship Bias Warning

The 9,744% Number Comes With a Giant Asterisk

The dynamic universe results are impressive, but they carry significant survivorship bias. The screening funnel uses current data availability to determine which tickers had sufficient history. Stocks that were delisted, went bankrupt, or were acquired during the backtest period may be excluded from the dynamic pool even though they would have been eligible at the time.

Look-Ahead Bias in Screening

The funnel knows which tickers have data through 2026. A real-time implementation would not have this information. Some tickers that passed the screen in backtesting would have been delisted before they could be traded in real time.

The 4.1x Multiplier Is Likely Inflated

We expect the real-world improvement from dynamic screening to be meaningful but substantially less than 4.1x. A conservative estimate, after correcting for survivorship and look-ahead bias, would be in the 1.5–2.5x range. We are currently building a point-in-time database to validate this estimate.

We publish these results with full transparency because intellectual honesty is a core value of the StockQueen project. The directional finding—that a larger, dynamically screened universe outperforms a static list—is almost certainly valid. The magnitude of the improvement is the part that needs further validation.

What's Next: ML-V3

The ML-V2 experiments revealed a clear path forward. The pairwise ranking approach works in stress environments but costs returns in bull markets. ML-V3 will address this with a regime-conditional architecture:

1

Regime-Aware ML

Train separate models for bull, bear, and transitional regimes. Only activate the ML layer when the current regime matches conditions where ML-V2 demonstrated positive alpha.

2

Point-in-Time Dynamic Universe

Build a survivorship-bias-free ticker database that only uses information available at each historical date, eliminating look-ahead bias from the dynamic screening funnel.

3

Ensemble Scoring

Combine momentum scores with ML rankings using a weighted ensemble, where the ML weight adapts based on recent out-of-sample performance.

ML for Offense. Dynamic Universe for Opportunity.

Two experimental tracks, one goal: find more winners without compromising the defense stack. ML-V2 shows promise in crash environments. Dynamic screening unlocks 3x more tickers. Both are active research—follow along as we iterate toward production readiness.
