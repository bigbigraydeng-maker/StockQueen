Regime 分类 (总分 -5~+5)

总分范围 Regime 标签 市场特征 策略转换
≥ 4 🟢 Strong Bull 强势上涨，低波动 激进进攻（70% V4）
1 ~ 3 🟡 Bull 温和上涨，MR 机会 均衡配置（60% V4 + 10% MR）
-1 ~ 0 🟠 Choppy 震荡市，反向机会 防守转攻（30% V4 + 50% MR + 10% Hedge）
≤ -2 🔴 Bear 下跌风险，保本第一 防守持仓（50% 现金 + 30% Hedge）

⏱️ 3 日确认机制

新 regime 需连续 3 天同向才正式切换。

目的：防止单日噪声触发无效轮动。

  • Day 1: 记录但保持旧 regime
  • Day 2: 再次确认
  • Day 3: 连续 3 天 → 正式切换

💾 进程内缓存

同一调度周期（≤30分钟）内，多次调用 _detect_regime() 返回同一结果。

优势：

  • 节省 Alpha Vantage API 调用
  • 避免行中数据不一致
  • 提升执行速度
