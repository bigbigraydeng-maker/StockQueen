# StockQueen CMS 系统设计
## 可视化内容管理系统 (Visual CMS)

---

## 🎯 设计理念

**核心原则**: 像使用 Notion 一样简单，像专业 CMS 一样强大

**解决 Sanity 的痛点**:
- ❌ Sanity: 学习曲线陡峭，配置复杂
- ✅ 我们的 CMS: 开箱即用，可视化操作，零配置

---

## 🏗️ 系统架构

### 技术栈
```yaml
前端:
  - Next.js 14 (App Router)
  - Tailwind CSS + shadcn/ui
  - TipTap 编辑器 (Notion 风格)
  - React DnD (拖拽排序)

后端:
  - Next.js API Routes
  - Prisma ORM
  - PostgreSQL (数据)
  - Redis (缓存/队列)

存储:
  - 本地文件系统 (开发)
  - AWS S3 / Cloudflare R2 (生产)
  - Git 自动同步

部署:
  - Vercel (推荐)
  - 或自托管
```

## 🛠️ 核心功能模块

### 1. 可视化编辑器 (Visual Editor)

**特性**:
- 📝 Notion 风格的块编辑器
- 🖱️ 拖拽排序
- 📊 一键插入 StockQueen 数据卡片
- 🎨 实时预览
- ⌨️ 快捷键支持

**块类型**:
```typescript
const blockTypes = [
  'heading',      // 标题
  'paragraph',    // 段落
  'image',        // 图片
  'data-card',    // 数据卡片
  'chart',        // 图表
  'quote',        // 引用
  'divider',      // 分割线
  'callout',      // 提示框
  'table',        // 表格
  'code',         // 代码块
  'embed',        // 嵌入内容
];
```

---

### 2. AI 内容工厂

**功能**:
- 🔥 热点新闻自动抓取
- ✍️ 一键改写为博客文章
- 📊 自动插入相关数据
- 🎯 SEO 自动优化
- 📝 多版本标题生成

**工作流程**:
```
热点新闻 → AI分析 → 改写草稿 → 人工审核 → 一键发布
                ↓
         自动生成社交媒体文案
         自动生成配图建议
         自动生成视频脚本
```

## 🚀 技术实现方案

### 方案 A: 基于 Next.js 的全栈应用 (推荐)

**优点**:
- 一体化开发，前后端统一
- 部署简单 (Vercel 一键部署)
- 性能优秀
- 生态丰富

**项目结构**:
```
cms/
├── app/
│   ├── (dashboard)/
│   │   ├── page.tsx              # 仪表盘
│   │   ├── editor/
│   │   │   └── page.tsx          # 编辑器
│   │   ├── calendar/
│   │   │   └── page.tsx          # 内容日历
│   │   ├── analytics/
│   │   │   └── page.tsx          # 数据分析
│   │   └── settings/
│   │       └── page.tsx          # 设置
│   ├── api/
│   │   ├── posts/
│   │   ├── ai/
│   │   ├── trends/
│   │   └── publish/
│   └── layout.tsx
├── components/
│   ├── editor/                   # 编辑器组件
│   ├── data-cards/              # 数据卡片
│   ├── ai-panel/                # AI面板
│   └── ui/                      # UI组件
├── lib/
│   ├── prisma.ts                # 数据库
│   ├── ai.ts                    # AI集成
│   └── github.ts                # Git同步
└── types/
    └── index.ts
```

### 方案 B: 无头 CMS + 自定义前端

**可选方案**:
- **Strapi**: 开源，自托管，GraphQL
- **Directus**: 开源，SQL直连，界面友好
- **Payload CMS**: Next.js原生，代码优先

**推荐**: Payload CMS (代码优先，易于定制)

## 💡 与 Sanity 对比

| 特性 | Sanity | 我们的 CMS |
|------|--------|-----------|
| 学习曲线 | 陡峭 | 平缓 |
| 配置复杂度 | 高 (需写 Schema) | 低 (可视化配置) |
| 数据集成 | 需开发 | 内置 StockQueen 数据 |
| AI 功能 | 无 | 内置 |
| 社交发布 | 需集成 | 内置 |
| 自托管 | 困难 | 简单 |
| 定制成本 | 高 | 低 |

---

## 🎯 推荐使用方案

**推荐**: 方案 A (Next.js 全栈)

**原因**:
1. 完全掌控，无供应商锁定
2. 深度集成 StockQueen 数据
3. 可定制 AI 工作流
4. 部署维护简单
5. 长期成本更低

**预计开发时间**: 6-8 周
