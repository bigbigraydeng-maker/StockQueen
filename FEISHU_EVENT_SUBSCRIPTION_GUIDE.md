## 配置步骤

### 1. 登录飞书开放平台

访问：https://open.feishu.cn/app

### 2. 选择你的应用

找到应用ID为 `cli_a92adfa4a478dbc2` 的应用

### 3. 配置权限

进入 **权限管理** 页面，确保已开通以下权限：

#### 必需权限：
- **获取与发送单聊、群组消息** (`im:message`)
- **读取用户发给机器人的单聊消息** (`im:message:send_as_bot`)
- **读取群组中@机器人的消息** (`im:message:group_at_msg`)
- **以机器人的身份发送消息** (`im:message:send_as_bot`)

### 4. 配置事件订阅

进入 **事件与回调** > **事件配置** 页面：

#### 步骤4.1：选择订阅方式
- 选择 **使用长连接接收事件**
- 点击 **保存**

#### 步骤4.2：添加事件
点击 **添加事件**，添加以下事件：

1. **接收消息事件** (v2.0)
   - 事件类型：`im.message.receive_v1`
   - 版本：v2.0
   - 描述：接收用户发给机器人的消息

2. **机器人被加入群组事件** (v1.0)
   - 事件类型：`im.chat.member.bot.added_v1`
   - 版本：v1.0
   - 描述：机器人被添加到群组

3. **机器人被移出群组事件** (v1.0)
   - 事件类型：`im.chat.member.bot.deleted_v1`
   - 版本：v1.0
   - 描述：机器人被移出群组

### 5. 发布应用

配置完成后，需要发布应用：

1. 进入 **版本管理与发布** 页面
2. 点击 **创建版本**
3. 填写版本信息
4. 点击 **发布**

### 6. 将机器人添加到群组

#### 方法1：通过飞书客户端
1. 打开飞书客户端
2. 进入要添加机器人的群组
3. 点击群组设置
4. 选择 **添加成员**
5. 搜索你的机器人名称
6. 添加到群组

#### 方法2：通过开放平台
1. 进入 **应用管理** > **机器人设置**
2. 复制机器人的分享链接
3. 将链接发送到群组中
4. 点击链接添加机器人

### 7. 测试机器人

在飞书中给机器人发送消息：

#### 测试命令：
- `status` 或 `状态` - 查看系统状态
- `signals` 或 `信号` - 查看当前信号
- `watchlist` 或 `关注` - 查看关注列表

2. 进入应用管理

4. 确认显示 **已建立长连接**

查看应用日志，确认：
- `[FeishuEventService] Feishu WebSocket connected successfully`
- `[FeishuEventService] Received message event v2.0`

## 常见问题

### Q1: 无法保存事件订阅方式

**原因**：本地客户端未启动或长连接未建立

**解决**：
1. 确保StockQueen应用正在运行
2. 查看日志确认长连接已建立
3. 等待3-5秒后再试

### Q2: 长连接已建立但收不到消息

**原因**：
- 权限未开通
- 事件未添加
- 机器人未添加到群组

**解决**：
1. 检查权限配置
2. 检查事件订阅配置
3. 将机器人添加到群组

### Q3: 机器人没有响应

**原因**：
- 消息处理逻辑有问题
- 发送消息权限未开通

**解决**：
1. 查看应用日志
2. 检查 `im:message:send_as_bot` 权限
3. 确认消息格式正确

## 技术细节

### 长连接模式优势

1. **开发成本低**：无需公网IP或域名
2. **安全性高**：内置加密传输
3. **部署简单**：本地开发环境即可使用

### 事件版本说明

- **v1.0事件**：使用 `register_p1_customized_event` 处理
- **v2.0事件**：使用 `register_p2_xxx` 处理

### 消息处理流程

1. 用户发送消息
2. 飞书服务器通过长连接推送事件
3. Lark SDK接收事件
4. 事件处理器处理消息
5. 调用API发送响应
