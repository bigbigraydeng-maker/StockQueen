# 已知问题与局限

---

## 回测层面

### 1. 幸存者偏差 (Survivorship Bias)
**严重度**: 中等
**状态**: V5 部分解决

| 版本  | 情况                       |
| --- | ------------------------ |
| V4  | 220只全是当前存活的股票，退市/破产公司未纳入 |
| V5  | 动态宇宙扫描，按日期过滤IPO/退市，大幅缓解  |

**残留风险**: 即使V5，也依赖AV LISTING_STATUS的退市数据完整性

### 2. 跳空风险 (Gap Risk)
**严重度**: 低
**状态**: 未解决

ATR止损假设能在止损价精确成交。实际隔夜跳空缺口（如财报后暴跌10%）会导致成交价远差于止损价。

**缓解**: ATR_STOP=1.5 包含一定缓冲。

### 3. 多轮测试数据窥探 (Data Snooping)
**严重度**: 低至中等
**状态**: 已量化

我们迭代了 V1→V4 共 4 轮，每轮看 OOS 结果后调方向。这引入轻度数据窥探。

**量化**: 过拟合衰减 0.23（V4）说明即使考虑此因素，edge 仍存在。

### 4. 极端市场覆盖不足
**严重度**: 不确定
**状态**: V5 计划解决

2021-2026 回测期无 2008 式金融危机或 2020.3 式恐慌。

**V5 方案**: Regime仓位管理（熊市自动降仓50%）+ 回测扩展到2018年

### 5. Fundamental 因子回测禁用
**严重度**: 低
**状态**: 已处理

Fundamental 因子使用 AV Overview 数据，有前视偏差风险（当前数据反映未来）。
回测中已禁用。实盘中启用。

**影响**: 回测可能低估实盘表现（实盘多了一个有效因子）

### 6. Sentiment 因子回测不可用
**严重度**: 低
**状态**: 已处理

AV NEWS_SENTIMENT 无历史归档，无法回测。回测中权重自动归零。

## 策略层面

### 7. 容量限制
**严重度**: 取决于资金规模
**状态**: 未解决

TOP_N=6 每周调仓，涉及频繁买卖。

| 资金规模 | 影响 |
|----------|------|
| < $50K | 无影响 |
| $50K - $500K | 轻微市场冲击 |
| > $500K | 需要分批执行，可能影响成交价 |
| > $1M | 建议增大 TOP_N 或分散到多账户 |

### 8. 均值回归仅牛市激活
**严重度**: 低
**状态**: 设计决策

震荡市最需要均值回归，但当前 `ACTIVE_REGIMES = {"bull"}`。

**原因**: choppy 时超卖可能是趋势下跌，胜率不够。
**可改进**: 未来可加入 choppy，配合更严格的入场条件。

### 9. 事件驱动只看财报
**严重度**: 低
**状态**: V5 计划扩展

当前事件驱动仅围绕 earnings。
V5 计划加入: FDA审批、分析师升降级、并购公告等。
