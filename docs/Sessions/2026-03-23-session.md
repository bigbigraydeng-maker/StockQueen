## 背景

## Phase 1: 交易安全修复（CRITICAL）

### 1.1 防止重复建仓 ✅

- **问题**: `_manage_positions_on_rotation` 用两次独立查询检查 pending_entry + active，insert 无事务保护，并发时可能双建仓
- **修复**: Supabase 添加 partial unique index `uq_rotation_active_ticker`，Python 端 catch duplicate key error
- **文件**: `rotation_service.py:3250-3265`
- **DB**: `CREATE UNIQUE INDEX uq_rotation_active_ticker ON rotation_positions(ticker) WHERE status IN ('pending_entry','active')`

### 1.2 下单前校验 buying_power ✅

- **问题**: `calculate_position_size` 只按 equity/max_positions 算仓位，不检查实际可用购买力，满仓时仍会尝试下单
- **修复**: 从 Tiger 账户资产中读取 buying_power，allocation 超出时自动缩减至 bp×0.95，并 log warning
- **文件**: `order_service.py:383-402`

### 1.3 滑点追踪 ✅

- **问题**: entry_price 被 Tiger fill price 覆盖后，原始信号价丢失，无法评估执行质量
- **修复**:
  - `rotation_positions` 表新增 `signal_price` 列
  - `_activate_position` 在下单前保存 signal_price
  - fill 后计算 `slippage_bps = (fill - signal) / signal × 10000` 并 log
- **文件**: `rotation_service.py:3349, 3399-3401`

### 1.4 订单审计日志表 ✅

- **问题**: 订单状态变更无持久化审计记录，出问题后无法复盘
- **修复**:
  - 新建 `order_audit_log` 表（position_id, ticker, action, order_type, quantity, signal_price, fill_price, slippage_bps, tiger_order_id, status, error_message, created_at）
  - 新增 `_log_order_audit()` 函数
  - 在 BUY submitted / filled / rejected / error 四种状态都写入审计记录
- **文件**: `order_service.py:21-50`, `rotation_service.py:3385-3430`

### 1.5 价格浮点精度修复 ✅

- **问题**: Alpha Vantage 股息调整价格用 raw float 乘法，无 rounding，累积浮点误差影响止损止盈计算
- **修复**: Open/High/Low/Close 统一 `round(..., 4)`
- **文件**: `alphavantage_client.py:308-312`

### 1.6 配置冲突修复 ✅

- **问题**: `RotationConfig.TOP_N = 3`，但 `RiskConfig.MAX_POSITIONS = 2`，实际 rotation 用 TOP_N，RiskConfig 被忽略
- **修复**: `MAX_POSITIONS` 改为 3，与 TOP_N 一致
- **文件**: `config/settings.py:122`

## Phase 2: 性能优化

### 2.1 N+1 查询改并发 ✅

- **问题**: `/htmx/pending-entries` 和 `/htmx/weekly-report` 逐个 ticker 串行调用 `_fetch_history()`
- **修复**: 改为 `asyncio.gather(*tasks, return_exceptions=True)` 并发获取
- **效果**: 3 个 pending ticker 从 ~3s 降至 ~1s
- **文件**: `web.py:1107-1125`, `web.py:2795-2816`

### 2.2 无界 DB 查询加 LIMIT ✅

- **问题**: `/api/public/signal-history` 和 `/api/public/paper-vs-wf` 取所有 closed trades 无 limit
- **修复**: 分别加 `.limit(200)` 和 `.limit(500)`
- **文件**: `web.py:3370, 3458`

### 2.3 API 超时优化 ✅

- **问题**: Alpha Vantage 和 Massive 客户端 httpx 超时 30s，阻塞 throttle lock 影响所有后续请求
- **修复**: 改为 `httpx.Timeout(connect=5.0, read=10.0, write=5.0, pool=5.0)`
- **文件**: `alphavantage_client.py:85`, `massive_client.py:99,1071`

### 2.4 内存泄漏清理 ✅

- **问题**: `_bt_jobs` dict 永不清理；`_cache` dict 过期条目不主动清除
- **修复**:
  - `_bt_jobs`: 每次创建新 job 时清理 >1h 的旧 job
  - `_cache`: 每 100 次 `_cache_get` 调用时批量清理过期 key
- **文件**: `web.py:2687-2693`, `web.py:92-100`

### 2.5 启动加速 ✅

- **问题**: `main.py` 启动时 `asyncio.sleep(10)` 等待服务器稳定
- **修复**: 改为 `asyncio.sleep(2)`
- **文件**: `main.py:121`

## Phase 3: 安全加固

### 3.1 CSP + HSTS 安全头 ✅

- **问题**: SecurityHeadersMiddleware 缺少 Content-Security-Policy 和 Strict-Transport-Security
- **修复**:
  - 添加 CSP: 限制 script-src/style-src/connect-src/img-src/font-src 来源
  - 添加 HSTS: `max-age=31536000; includeSubDomains`
- **文件**: `main.py:219-230`

### 3.2 依赖版本锁定 ✅

- **问题**: pydantic、pydantic-settings、Pillow、tigeropen 等未锁定版本
- **修复**: 锁定为 pydantic==2.10.6, pydantic-settings==2.7.1, Pillow==10.4.0, tigeropen==3.4.0 等
- **文件**: `requirements.txt`
- **注意**: lark-oapi==1.3.16 不存在，回退为 `>=1.3.0`
